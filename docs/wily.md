# Wily

Python 코드의 복잡성을 체크하는 명령어 도구

### 1. 순환 복잡도 (Cyclomatic Complexity):

- 코드의 제어 흐름 그래프에서 독립적인 경로의 수로 코드의 복잡성을 측정합니다.
- negie Mellon University의 Software Engineering Institute는 순환 복잡도를 기반으로 위험 수준을 구분합니다

| 점수  | 설명                                        |
| ----- | ------------------------------------------- |
| 1–10  | 위험이 적은 간단한 프로그램                 |
| 11–20 | 중간 위험                                   |
| 21–50 | 복잡하고 높은 위험이 있는 프로그램          |
| > 50  | 테스트하기 어려운 매우 높은 위험의 프로그램 |

### 2. Halstead 복잡도 측정 (Halstead Complexity Measures):

- 연산자와 피연산자의 수를 바탕으로 프로그램의 난이도, 노력, 구현 시간을 계산하는 정적 분석 절차입니다.
- 높을수록 복잡하다.

### 3. 유지보수성 지수 (Maintainability Index):

- 순환 복잡도, Halstead 복잡도 측정, 코드 라인 수를 기반으로 평가한 종합 점수입니다.

| 점수   | 설명            |
| ------ | --------------- |
| 0-25   | 유지보수 불가능 |
| 25–50  | 우려스러움      |
| 50–75  | 개선이 필요함   |
| 75–100 | 슈퍼히어로 코드 |
